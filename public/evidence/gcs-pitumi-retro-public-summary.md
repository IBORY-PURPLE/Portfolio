# GCS8 1차 피투미 고객 검증 회고 - 공개 요약

> 이 문서는 포트폴리오 외부 공개용 요약입니다. 음성 전사 원문, 개인 실명, 직접 발화, 내부 평가와 팀원별 기록은 포함하지 않습니다.

## 검증 맥락

선원·해운업 종사자가 겪는 문제를 정의하기 위해 고객 인터뷰와 발표 실험을 진행했습니다. 초기 가설은 충분한 데스크 리서치와 타깃 분리 없이 시작되어, 인터뷰를 문제 검증보다 문제 발견에 사용하는 한계가 있었습니다.

## 확인된 수치

| 항목 | 확인 범위 |
| --- | --- |
| 인터뷰 | 총 8건 |
| 실제 핵심 타깃 | 3~4건 |
| 유사 타깃 | 4~5건 |
| 문제 정의 수정 | 3회 |
| 명시적 5 Whys 실행 | 0회 |

## 발견한 문제

- 실제 핵심 타깃과 유사 타깃을 사전에 구분하지 않아 인터뷰 결과의 해석이 흐려졌습니다.
- 논문, 법·정책, 경쟁사와 기존 업무 프로세스 조사가 충분하지 않은 상태에서 인터뷰를 시작했습니다.
- 질문지가 초기 가설의 원인을 깊게 검증하기보다 넓은 어려움을 확인하는 수준에 머물렀습니다.
- 개인별 기여와 의사결정 근거를 분리해 기록하지 않아 사후 역할 검증에 한계가 있었습니다.

## 다음 실험 기준

1. 인터뷰 전에 논문, 정책, 경쟁사, 기존 프로세스를 확인하는 리서치 게이트를 둡니다.
2. 핵심 타깃과 유사 타깃의 기준을 정의하고 인터뷰 수를 분리해 기록합니다.
3. 최소 한 번의 5 Whys와 사전 질문 검토를 통과한 뒤 고객 인터뷰를 시작합니다.
4. 인터뷰 다수가 문제를 부정하면 가설을 폐기하는 Kill Rule을 사전에 합의합니다.
5. 결정 근거와 개인별 실행 범위를 주 단위로 기록합니다.

## 포트폴리오에서 사용하는 근거

이 사례는 성공 성과가 아니라 약한 가설을 발견하고 다음 검증 방식을 개선한 학습 근거로 사용합니다. 인터뷰 수와 학습 기준 외의 개인 발화, 내부 평가, 실명 자료는 공개 근거로 사용하지 않습니다.
